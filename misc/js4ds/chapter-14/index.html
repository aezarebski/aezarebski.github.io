Other Data Formats

The following code will read in the data and then uses the parseInts method to parse the values as integers. After being written and read from JSON, data-forge knows that these values should be integers rather than strings. 

// demo.js
const fs = require('fs');
const DF = require('data-forge');

const inputCsv = 'demo.csv';
const outputJson = 'demo.json';
const outputCsv = 'demo-again.csv';

const text = fs.readFileSync(inputCsv, 'utf-8');
const colours = DF.fromCSV(text).parseInts(['red', 'green', 'blue']);

fs.writeFileSync(outputJson, colours.toJSON());

const textAgain = fs.readFileSync(outputJson, 'utf-8');
const coloursAgain = DF.fromJSON(textAgain);

Directional Sorting

The following reads in the colour data as before and then sorts it, first by increasing values of red and then by decreasing values of green. 

const fs = require('fs');
const DF = require('data-forge');

const inputCsv = 'demo.csv';

const text = fs.readFileSync(inputCsv, 'utf-8');
const colours = DF.fromCSV(text).parseInts(['red', 'green', 'blue']);

const sortedColours = colours
      .orderBy(row => row.red)
      .thenBy(row => -row.green);

console.log(sortedColours.toArray());

Multi-Column Distinct

Rather than comparing the rows as a list of numbers we can combine the numbers as a single string and use these as the comparison of equality. 

const multiColumn = colours
      .distinct(row => String(row.red) + String(row.green))
      .orderBy(row => row.red)
      .thenBy(row => row.green)
console.log(multiColumn.toArray())

Revisiting Data Manipulation

We can summarise the desired columns with the following script. 

const fs = require('fs');
const dataForge = require('data-forge');

const inputCsv = 'surveys.csv';

const text = fs.readFileSync(inputCsv, 'utf-8');
const surveys = dataForge.fromCSV(text)
      .parseInts(
          ['record_id',
           'month',
           'day',
           'year',
           'plot_id',
           'hindfoot_length',
           'weight']);

const seriesSummary = (seriesName) => {
    const series = surveys.getSeries(seriesName);
    return {
        series: seriesName,
        min: series.min(),
        mean: series.mean(),
        max: series.max()
    };
};

const summary = new dataForge.DataFrame({
    values: ['year', 'hindfoot_length', 'weight'].map(seriesSummary)
});

console.log(summary.toCSV());

Which when run produces the following output.

year,1977,1990.475231370784,2002
hindfoot_length,2,29.287931802277498,70
weight,4,42.672428212991356,280

Grouping and Aggregating

The relevant code is the following 

const averaged = earthquakes
      .groupBy(row => row.Magnitude)
      .select(group => ({
        Magnitude: group.first().Magnitude,
        Depth_Km: group.deflate(row => row.Depth_Km).average()
      }))
      .inflate()
      .orderBy(row => row.Magnitude)

The documentation says many things, we must be careful about the return type of the functions since this will determine which method gets called.

  • groupBy takes a dataframe to a series of dataframes,
  • when applied to a series, select generates a new series
  • inflate will then turn the series into a dataframe, and
  • orderBy will sort the rows of the dataframe.

Note that the deflate function does the reverse of inflate, it takes a dataframe to a series.

Not a Number

The following takes the records with either a year or a visitor count that cannot be parsed to an integer and prints out some relevant information. 

const fs = require('fs');
const dataForge = require('data-forge');


const csvString = fs.readFileSync('national_parks.csv', 'utf-8');
const parks = dataForge.fromCSV(csvString)
      .subset(['year', 'visitors']);

console.log('The broken values in the year column all contain the same value:');
console.log(parks.generateSeries({
    yearInt: row => row.year,
    visitorsInt: row => row.visitors
})
            .parseInts(['yearInt', 'visitorsInt'])
            .where(row => isNaN(row.yearInt))
            .getSeries('year')
            .distinct()
            .toArray());

console.log('There are only a few lines with broken visitor numbers:');
console.log(parks.generateSeries({
    yearInt: row => row.year,
    visitorsInt: row => row.visitors
})
            .parseInts(['yearInt', 'visitorsInt'])
            .where(row => isNaN(row.visitorsInt))
            .toArray());
