What Encoding Is This?

To log the character set of the current page you would just need to console.log the variable document.charset. If you insisted on looking up the meta tags and looking this up directly there is the following function 

const logEncoding = () => {
    let els = Array.from(document.getElementsByTagName("meta"))
        .filter((x) => x.hasAttribute("charset"));
    console.log("The encoding on this page is " + els[0].getAttribute("charset"));
};

Word Count

To count the total number of "words" in the text of the nodes on a page there is the innerText of the body which seems to be sufficient. We need to do a little bit of extra work to ensure that new lines and repeated spaces are handled properly. 

const countWords = () => {
    let lines = document
        .getElementsByTagName("body")[0]
        .innerText
        .split("\n")
        .filter((x) => {return x !== "";});
    let n = 0;
    for (var i = 0; i < lines.length; i++) {
        n += lines[i]
            .split(" ")
            .filter((x) => {return x !== "";})
            .length;
    }
    return n;
};

A function that adds this information to the page might look like this 

const showWords = () => {
    const newPara = document.createElement("p");
    newPara.setAttribute("id", "wordcount");
    const newContent = document.createTextNode(`There are ${countWords()} words.`);
    newPara.appendChild(newContent);
    document.body.appendChild(newPara);
}

A More Robust Table of Contents

If there is no id attribute, the table of contents' link will not function properly. If we wanted to exclude h2 elements that do not have an id attribute then we would need to add a filter to the headings. This could be achieved by defining items in the following way: 

const items = headings
      .filter((h) => h.hasAttribute("id"))
      .map((h) => `<li><a href="#${h.id}">${h.innerHTML}</a></li>`)
      .join('');

Explicitly Creating Nodes

document.createElement and document.createTextNode can be used to create the table of contents in a neater way. In the case of the link though, I still use the innerHTML approach to ensure that the styling is preserved. 

const listElement = (id,text) => {
    const link = document.createElement("a");
    link.setAttribute("href", `#${id}`);
    link.innerHTML = text;
    const entry = document.createElement("li");
    entry.appendChild(link);
    return entry;
};

(() => {
    const container = document.getElementById('fill');
    const headings = Array.from(document.querySelectorAll('h2'));
    const table = document.createElement("ul");
    const items = headings
          .filter((h) => h.hasAttribute("id"))
          .map((h) => table.appendChild(listElement(h.id, h.innerHTML)));
    container.appendChild(table);
})();
