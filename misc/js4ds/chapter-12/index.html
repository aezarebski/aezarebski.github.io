Creating a Server

Set up

Start by setting up a project with express and a directory, web-dir, to serve files from with static-page.js. We include the secret file web-dir/secret.txt to have something to test permissions with. 

mkdir cas
cd cas
npm init -y
npm install express
touch static-page.js
# write static-page.js...
mkdir web-dir
echo "<html><body><h1>Home</h1></body></html>" > web-dir/home.html
touch web-dir/secret.txt
chmod -r web-dir/secret.txt
# copy a PNG file into web-dir as well.
node static-page.js ./web-dir

The script is 

const express = require('express');
const path = require('path');
const fs = require('fs');

const PORT = 3418;
const root = process.argv[2];
const app = express();

app.use((req, res, next) => {
    const actual = path.join(root, req.url);

    if (actual.endsWith('.json')) {
        const data = fs.readFileSync(actual, 'utf-8');
        const json = JSON.parse(data);
        res.setHeader('Content-Type', 'application/json');
        res.status(200).send(json);
    }

    else {
        const data = fs.readFileSync(actual, 'utf-8');
        res.status(200).send(data);
    }
});

app.listen(PORT, () => { console.log('listening...') });

Some pages to visit include

  • http://localhost:3418/home.html which will work,
  • http://localhost:3418/fleeb.html which will not work, and
  • http://localhost:3418/secret.txt which will not work.

Exercises

Report Missing Files

From the documentation we can see that the way to handle missing files and permissions is to use the readFile function with a proper call back. They explicitly advice against using other methods to get this information as it can lead to race problems. The solution then is to replace 

const data = fs.readFileSync(actual, 'utf-8');
res.status(200).send(data);

with 

fs.readFile(actual, 'utf-8', (err, data) => {
    if (err) {
        if (err.code == 'ENOENT') {
            res.status(404).send(`<html><body><p>ERROR: ${req.url} not found</p></body></html>`);
        }
        if (err.code == 'EACCES') {
            res.status(401).send(`<html><body><p>ERROR: ${req.url} not authorised</p></body></html>`);
        }
    } else {
        res.status(200).send(data);
    }
});

Note that a similar change should be made for the JSON data too.

Serving Images

We can pattern match on a request for a PNG file and then send the contents of the file (which needs to be given as an absolute path for some reason), by adding the following branch after the match for a JSON file. 

// branch for JSON as before...

else if (actual.endsWith('.png')) {
    const absPath = path.join(__dirname, actual);
    res.setHeader('Content-Type', 'image/png');
    res.status(200).sendFile(absPath);
}

Using Query Parameters

The following can be added to the top of the script to parse the mathematical problem URLs. 

app.get('/math/add', (req, res, next) => {
    const result = parseInt(req.query.left)
          + parseInt(req.query.right);
    res.setHeader('Content-Type', 'application/json');
    res.status(200).send({result: result});
});

app.get('/math/multiply', (req, res, next) => {
    const result = parseInt(req.query.left)
          * parseInt(req.query.right);
    res.setHeader('Content-Type', 'application/json');
    res.status(200).send({result: result});
});
