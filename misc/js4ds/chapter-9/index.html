A Synchronous or Asynchronous?

Finding the nearest town and all of the routes between two locations seem like the kind of things that would require querying a database or API so it seems like they should run asynchronously, however computing the volume of a sphere is a simple expression so should be synchronous. Handling Exceptions

setTimeout puts the call to reject at the end of the queue and allows resolve('') to be evaluated in the mean time. Since this calls the checkForBlanks function this the error message is encountered. However, for this to actually be logged, it seems to propagate all the way through the error branches. To see this I had to test out using a string in place of error.message and checkedOutput.

Empty Promises

It looks like the syntax of setTimeout may have changed since the exercise was written, but the following worked for me. 

const makePromise = (someInteger) => {
    return new Promise((resolve, reject) => {
        setTimeout(resolve, someInteger * 1000, someInteger)
    })
}

Promise.all([makePromise(7), makePromise(8), makePromise(2),
             makePromise(6), makePromise(5)]).then(
                 numbers => console.log(numbers))

In order to print 2 instead of the whole array you can replace all with race so it just takes the result that was returned first which in this case is the 2.

async, Therefore I Am

I cannot be sure, but I think the way to convert makePromise into an async function is to change it to the following 

const makePromise = async (someInteger) => {
    let foo = new Promise((resolve, reject) => {
        setTimeout(resolve, someInteger * 100, someInteger)
    });
    return await foo;
}
