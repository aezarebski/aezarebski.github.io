Managing data

Set up

mkdir managing-data
cd managing-data
npm init -y
npm install papaparse seedrandom
touch select-random.js
node select-random.js surveys.csv 10 slice.csv

The starting form of select-random.js is the following 

const fs = require('fs');

const [inputFile, numLines, outputFile] = process.argv.splice(2);
const lines = fs.readFileSync(inputFile, 'utf-8')
      .split('\n');
header = lines[0];
const sample = lines.slice(1)
      .map(line => [Math.random(), line])
      .sort((left, right) => { return left[0] - right[0]; })
      .slice(0, parseInt(numLines))
      .map(pair => pair[1]);
fs.writeFileSync(outputFile, header + '\n' + sample.join('\n'));

Exercises

Tracing Data

const sample = lines.slice(1)
      .map(line => [Math.random(), line])
      .sort((left, right) => { return left[0] - right[0]; })
      .slice(0, parseInt(numLines))
      .map(pair => pair[1]);
  1. Slice off the first line (which is the header).
  2. Combine each line with a random number.
  3. Sort the lines based on the random numbers.
  4. Take the desired number of lines from the sorted lines.
  5. Remove the random number so only the lines remain.

Unrandom

We had already installed seedrandom as part of the set up. To import the package and use a command line argument as the seed for the PRNG the first lines of select-random.js become the following: 

const seedrandom = require('seedrandom');
const fs = require('fs');

const [inputFile, numLines, outputFile, seed] = process.argv.splice(2);
var prng = seedrandom(seed);

and then Math.random() needs to be replaced with prng() to make use of the new sampler.

Then we read the name of the desired file from the command line arguments and construct a data manager object from that. 

const [inputFile] = process.argv.splice(2);
const records = new DataManager(inputFile);

The insertStatement is a pure function which returns a statement to insert this record into a SQL table called "surveys". 

const insertStatement = (record) => {
    const keys = Object.keys(record).filter((k) => !(k === null));
    const vals = keys.map((k) => record[k]);
    if (vals.length > 0) {
        return `INSERT INTO surverys (${keys}) VALUES (${vals})`;
    }
};

Finally we can create a pipeline in which the array of records is reduced with a function that accumulates records from unique years and then logs the insertion statement for each of these records. 

records

Error Handling

We can expand the DataManager class to have a list of parse errors (which are returned by papaparse) and other errors which may be beyond the scope of the parsing step. 

class DataManager {

    constructor (filename) {
        this.parseErrors = [];
        this.otherErrors = [];

        if (fs.existsSync(filename)) {
            const raw = fs.readFileSync(filename, 'utf-8');
            const options = {header: true, dynamicTyping: true};
            const parseResult = papa.parse(raw, options);
            this.data = parseResult.data;
            this.header = parseResult.meta.fields;
            this.parseErrors = parseResult.errors;
        } else {
            this.otherErrors = ['file not found error'];
        }
    }

}

Generalization

We can achieve the summary for arbitrary variables by using the spread syntax to collect them into an array and then construct the key-value pairs in a programmatic way. The only change is to the getSurveyRange method. 

getSurveyRange (minYear, maxYear, ...variables) {
    return Array(1 + maxYear - minYear)
        .fill(0)
        .map((v, i) => minYear + i)
        .map(year => {
            const subset = this.data.filter(r => r.year === year);
            if (subset.length) {
                const tmp = {year : year};
                for (var v of variables) {
                    tmp['min_' + v] = this._get(subset, v, Math.min);
                    tmp['ave_' + v] = this._get(subset, v, _average);
                    tmp['max_' + v] = this._get(subset, v, Math.max);
                }
                return tmp;
            }
        });
}
