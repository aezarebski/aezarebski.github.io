Reserved Symbols

Because certain characters are used by the XML syntax we need entities to represent them.

Entity Character
&lt; <
&gt; >
&dquot; "
&quot; '
&amp; &

Declaration

The near universal declaration at the top of an XML 

<?xml version='1.0' encoding='UTF-8' standalone='no'?>

The version is essentially redundant because everyone uses 1.0, the encoding might change and the standalone aspect is usually going to be 'no' because how often is a document completely self-contained?

Transformation with XSLT

XSLT is a language (itself using the XML syntax) which can be used to specify the transformation of an XML document. There are several implementations: libxslt and xalan look like they would be good starting points. There is also a xslt package from ROpenSci which provides a simple API for running these transformations.

Examples

Removing nodes

This example is taken from a StackOverflow answer. Consider the following data file. Suppose that we want to remove any nodes with the names username or password. This could be achieved using the XSLT file below, which when run creates the resulting XML document. 

<?xml version="1.0" ?>
<mydoc>
    <file>
        <colors>
            <blue />
            <red />
            <green />
        </colors>
        <secret>
            <username />
            <password />
        </secret>
    </file>
</mydoc>

<?xml version="1.0" encoding="UTF-8"?>
<xsl:stylesheet version="1.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">
    <xsl:template match="node() | @*">
        <xsl:copy>
            <xsl:apply-templates select="node() | @*"/>
        </xsl:copy>
    </xsl:template>
    <xsl:template match="username|password"/> <!-- this empty template will remove them -->
</xsl:stylesheet>

<?xml version="1.0" encoding="UTF-8"?>
<mydoc>
  <file>
    <colors>
      <blue/>
      <red/>
      <green/>
    </colors>
  </file>
</mydoc>

Linting and Formatting

Having a linter for XML is helpful. The tool js-beautify seems decent: LINK and GitHub. You can configure this if you do not like the default style it uses. Just add the following as .jsbeautifyrc in the root of your project. 

{
    "indent_size": 4,
    "preserve_newlines": true,
    "wrap_attributes": "force-aligned"
}
